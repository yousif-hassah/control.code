# حل مشكلة 401 Unauthorized - مفتاح Supabase مفقود

## المشكلة

التطبيق يعرض خطأ "Failed to create group" وأخطاء 401 في Console لأن مفتاح Supabase غير موجود في ملف `.env`

## الحل - خطوات بسيطة

### الخطوة 1: احصل على مفتاح Supabase

1. افتح متصفحك واذهب إلى: https://supabase.com/dashboard
2. سجل دخولك إذا لم تكن مسجلاً
3. ابحث عن المشروع الذي يحتوي على الرابط: `sqhflhwenjxkqwjlfxxi.supabase.co`
4. اضغط على المشروع لفتحه
5. من القائمة الجانبية، اضغط على **Settings** (الإعدادات) - أيقونة الترس ⚙️
6. اضغط على **API**
7. ستجد قسم اسمه **Project API keys**
8. انسخ المفتاح الموجود تحت **anon public** (يبدأ بـ `eyJ...`)

### الخطوة 2: حدّث ملف .env

1. افتح ملف `.env` في مجلد المشروع
2. ابحث عن السطر:
   ```
   VITE_SUPABASE_ANON_KEY=YOUR_ACTUAL_SUPABASE_ANON_KEY_HERE_STARTS_WITH_eyJ
   ```
3. استبدله بـ:
   ```
   VITE_SUPABASE_ANON_KEY=eyJ... (المفتاح الذي نسخته)
   ```

### الخطوة 3: أعد تشغيل السيرفر

1. في Terminal، اضغط `Ctrl+C` لإيقاف السيرفر
2. شغّل السيرفر مرة أخرى:
   ```bash
   npm run dev
   ```

### الخطوة 4: جرّب التطبيق

افتح التطبيق في المتصفح وحاول إنشاء مجموعة جديدة - يجب أن يعمل الآن! ✅

## ملاحظة مهمة

⚠️ **لا تشارك هذا المفتاح مع أحد!** هو مفتاح عام لكن يجب أن يبقى آمناً.

## إذا استمرت المشكلة

تأكد من:

- ✅ المفتاح يبدأ بـ `eyJ`
- ✅ لا توجد مسافات قبل أو بعد المفتاح
- ✅ تم إعادة تشغيل السيرفر بعد التحديث
